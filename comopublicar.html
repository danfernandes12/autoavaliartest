Guia: Como Publicar seu Site HTML no GitHub Pages

O GitHub Pages é um serviço gratuito para hospedar um site estático diretamente de um repositório.

Passo 1: Ter uma Conta no GitHub

Se você ainda não tem uma conta, precisará criar uma gratuitamente. É o primeiro passo para começar a usar todos os recursos do GitHub, incluindo o Pages.

Criar conta em github.com

Passo 2: Criar um Novo Repositório

  1. No canto superior direito, clique em + e selecione "New repository".
  2. Dê um nome ao seu repositório (ex: meu-site-onboarding).
  3. Mantenha o repositório como Público para usar o Pages gratuitamente.
  4. Clique em "Create repository".

Passo 3: Enviar seu Arquivo HTML

Agora, adicione seu arquivo HTML ao novo repositório.

  1. Clique em "Add file" e depois em "Upload files".
  2. Arraste e solte seu arquivo na área indicada.
  3. IMPORTANTE: O arquivo principal do seu site precisa se chamar index.html para que o GitHub Pages o reconheça automaticamente.

Dica!

Após o upload, você pode renomear o arquivo diretamente no GitHub clicando nele e depois no ícone de lápis (editar).

Passo 4: Ativar o GitHub Pages

  1. No seu repositório, clique na aba "Settings".
  2. Na barra lateral, clique em "Pages".
  3. Em "Source", selecione o branch de onde publicar (geralmente main).
  4. Deixe a pasta como /root e clique em "Save".

Passo 5: Acessar seu Site

O GitHub pode levar alguns minutos para publicar seu site. Após a conclusão, um link aparecerá na mesma seção (Settings > Pages).

Seu link terá este formato:

https://<seu-usuario>.github.io/<repositorio>/
